# 🎓 نظام إدارة مركز تعليمي (Education Center Management System)

هذا المشروع هو منصة تعليمية متكاملة مصممة لإدارة تجربة الطلاب والمعلمين والإداريين في مركز تعليمي.
تم بناء النظام باستخدام **NestJS** في الباك إند و**React + Vite** في الواجهة الأمامية، ويعمل مع **MySQL** عبر **Prisma ORM**.

---

## 🌟 ملخص المشروع

نظام متكامل يوفّر:
- تسجيل مستخدمين بطوائف مختلفة: طالب، معلم، وأدمن.
- إدارة وتحرير الكورسات والفصول التعليمية.
- نظام طلبات الالتحاق مع دورة دفع متكاملة.
- دعم الدفع الإلكتروني عبر بوابة Paymob، مع Webhook للتأكيد التلقائي.
- دعم نوعي الدفع: دفع مرة واحدة أو اشتراك شهري.
- تتبع الحضور والغياب مع حالات الحضور المختلفة.
- تنظيم الجدول الدراسي وإدارة القاعات مع الكشف عن تداخل المواعيد.
- إنشاء وإدارة الواجبات، رفع الحلول، وتصحيح الدرجات.
- إنشاء وإدارة الاختبارات وأسئلة الاختيار المتعدد.
- تتبع تقدم الطلاب داخل الفصول والكورسات.
- دعم رفع ملفات وسائط مثل الفيديوهات وPDF.
- وحدة دردشة داخلية ورسائل بريد إلكتروني لإشعارات الدفع والتذكير.

## 🚀 المميزات الرئيسية

### 1. المستخدمين والصلاحيات (Authentication & Authorization)
- تسجيل دخول وتسجيل حساب جديد.
- مصادقة JWT لحماية البيانات وواجهات الـ API.
- أدوار متعددة: طالب، معلم، ومدير.
- حماية الوصول للموارد حسب الدور.
- إدارة المستخدمين (عرض، تعديل، حذف، بحث).

### 2. إدارة الكورسات والمحتوى
- إنشاء الكورسات، تعديلها، وحذفها.
- تنظيم المحتوى في فصول (`Chapters`).
- عرض تفاصيل الكورس، وصفه، والأسعار.
- دعم رفع المواد التعليمية والوسائط عبر وحدة `upload-file`.
- دعم عرض الفيديو والنص وملفات PDF في المحتوى.

### 3. نظام الالتحاق والاشتراك
- تقديم طلب التحاق بالكورسات.
- عرض حالة طلب الالتحاق: `Pending`, `Wait for Payment`, `Approved`.
- ربط حالة الطلب بحالة الدفع.
- إدارة الطلبات للطالب والمعلم، وعرض سجلّ التحاق المستخدم.

### 4. الدفع الإلكتروني وإدارة الاشتراكات
- دمج مع بوابة الدفع Paymob.
- إنشاء طلبات دفع، إعادة توجيه المستخدم إلى صفحة الدفع، واستقبال Webhook للتأكيد.
- دعم نوعي الدفع: `ONE_TIME` و `MONTHLY`.
- إنشاء اشتراكات شهرية تلقائياً عند الدفع الناجح.
- جدول مخصص لدفعات الاشتراك الشهري وتنبيهات الدفع.

### 5. الجدول الدراسي وإدارة القاعات
- إضافة مواعيد الدروس لكورسات متعددة.
- إدارة القاعات وسعتها وحالتها.
- منع تداخل المواعيد داخل نفس القاعة.
- عرض جدول محاضرات منظم لكل كورس.

### 6. الحضور والغياب
- تسجيل حضور الطلاب لكل محاضرة.
- دعم حالات: حاضر، غائب، متأخر.
- توليد تقارير الحضور لكل طالب وكورس.
- عرض نسب حضور مع بيانات تحليلية.

### 7. الواجبات والتقييم
- إنشاء واجبات وربطها بكورسات وفصول محددة.
- رفع الطلاب لحلول الواجبات وتخزينها.
- تصحيح الواجبات وإضافة الدرجة والملاحظات.
- عرض الواجبات المقدمة وغير المقدمة في الواجهة الأمامية.

### 8. نظام الاختبارات
- إنشاء اختبارات وأسئلة اختيار متعدد.
- حفظ خيارات الأسئلة وربطها بالاختبارات.
- تسجيل محاولات الطلاب للامتحانات.
- عرض نتائج المحاولات وإحصائياتها.

### 9. تتبع التقدم وتقارير الطلاب
- تتبع تقدم الطالب داخل الفصول والكورسات.
- حساب نسبة الإنجاز لكل كورس.
- شاشة إحصائيات الطالب الحية بالاعتماد على بيانات الباك إند.
- ملخصات أداء للطلاب تشمل الحضور، الواجبات، والاختبارات.

### 10. تحميل الملفات والوسائط
- رفع ملفات الصور والفيديو والمستندات.
- تخزين الملفات ضمن نظام الباك إند.
- استخدام ملفات PDF ووسائط الفيديو في محتوى الكورس.

### 11. التواصل الداخلي والإشعارات
- وحدة `chat` لدعم المحادثات الداخلية بين المستخدمين.
- إرسال رسائل بريد إلكتروني تلقائية للتذكير بالدفعات الشهرية.
- دعم المراسلات الإدارية وتنبيهات النظام.

### 12. واجهة أمامية عربية متجاوبة
- واجهة مستخدم عربية كاملة.
- دعم RTL والتخطيط المتجاوب.
- صفحات خاصة بالدخول، إدارة الكورسات، المستخدمين، الفصول، والدفع.

## 🧠 تقنية البنية (Architecture)

### Backend
- `src/` يحتوي على كود Back-end باستخدام **NestJS**.
- البنية منظمة في وحدات (Modules) واضحة لكل وظيفة:
  - `auth`, `user`, `course`, `chapter`, `enrollment`, `chapter-progress`, `course-schedule`, `attendance`, `assignment`, `quiz`, `quiz-question`, `quiz-option`, `quiz-attempt`, `room`, `payments`, `chat`, `mail`, `upload-file`, `prisma`.
- `AppModule` في `src/app.module.ts` يجمع كل الوحدات ويربطها معاً.
- `src/main.ts` يهيء التطبيق ويشغّل:
  - `Swagger` في المسار `/api`.
  - `ValidationPipe` عام للتحقق من صحة البيانات.
  - استضافة الملفات الثابتة من مجلد `public`.
  - جدولة دفعة شهرية آلية عبر `PaymentsService` تعمل عند الإقلاع ثم كل 24 ساعة.
- سياسة الأمان:
  - حماية JWT في `auth`.
  - حماية المسارات حسب الدور عبر `RolesGuard` و `OwnershipGuard`.
  - مصادقة باستخدام `Passport` و `JwtStrategy`.

### Frontend
- `front end/` يحتوي على مشروع **React + Vite**.
- واجهة عربية متوافقة مع RTL.
- تستخدم `React Router` و `Axios` للتواصل مع API الباك إند.
- تحتوي على صفحات دخول، تسجيل، إدارة الكورسات والفصول، الدفع، التقارير، والإحصائيات.

### قاعدة البيانات
- `Prisma` يستخدم للتفاعل مع `MySQL`.
- مخطط البيانات في `prisma/schema.prisma`.
- ملفات الهجرات في `prisma/migrations/`.
- بيانات أولية يمكن تثبيتها من `mysql-init/`.

### إعدادات الباك إند الأساسية
- `JWT_SECRET` — مفتاح التوقيع لتوكنات JWT.
- `DATABASE_URL` — عنوان اتصال MySQL.
- `PORT` — رقم بورت تشغيل السيرفر (افتراضي `3000`).
- أسماء المستخدمين والأدوار تعتمد على مخطط `Role` الموجود في Prisma.

### تشغيل الـ API وثائق Swagger
- يتم تشغيل API على `http://localhost:3001` عند استخدام Docker.
- وثائق API متاحة في `http://localhost:3001/api`.

## ⚙️ كيفية التشغيل

### التشغيل باستخدام Docker (موصى به)
1. افتح Terminal في مجلد المشروع.
2. شغّل:
```bash
docker-compose up --build
```
3. افتح المتصفح:
- الواجهة الأمامية: `http://localhost:5173`
- الـ API: `http://localhost:3001`
- وثائق Swagger: `http://localhost:3001/api`
- phpMyAdmin: `http://localhost:8080`

### إيقاف الخدمة
```bash
docker-compose down
```

### تشغيل محلي بدون Docker
1. باك إند:
```bash
npm install
npm run start:dev
```
2. واجهة أمامية:
```bash
cd "front end"
npm install
npm run dev
```

> تأكد من وجود قاعدة بيانات MySQL متصلة وتحديث إعدادات `.env`، وتعيين `JWT_SECRET` و `DATABASE_URL` قبل التشغيل.
