# 🎓 نظام إدارة مركز تعليمي (Education Center Management System)

هذا المشروع هو منصة تعليمية متكاملة مصممة لإدارة تجربة الطلاب والمعلمين والإداريين في مركز تعليمي.
تم بناء النظام باستخدام **NestJS** في الباك إند و**React + Vite** في الواجهة الأمامية، ويعمل مع **MySQL** عبر **Prisma ORM**.

---

## 🌟 ملخص المشروع

نظام متكامل يوفّر:
- تسجيل مستخدمين بطوائف مختلفة: طالب، معلم، وأدمن.
- إنشاء الكورسات والفصول التعليمية.
- نظام طلبات الالتحاق ودفع رسوم الكورسات.
- تتبع الحضور والغياب.
- تنظيم جدول المحاضرات والقاعات.
- واجهة رفع وحل الواجبات.
- إنشاء وإدارة اختبارات عبر شبكة الإنترنت.
- تتبع تقدم الطالب في الكورس.
- دعم رفع ملفات الوسائط مثل الفيديوهات وPDF.
- إمكانية التواصل الداخلي والبريد الإلكتروني.

---

## 🚀 المميزات الرئيسية

### 1. المستخدمين والصلاحيات (Authentication & Authorization)
- تسجيل دخول وتسجيل حساب جديد.
- مصادقة JWT لحماية البيانات وواجهات الـ API.
- أدوار متعددة: طالب، معلم، ومدير.
- حماية الوصول للموارد حسب الدور.

### 2. إدارة الكورسات والمحتوى
- إنشاء وتعديل وحذف الكورسات.
- تنظيم المحتوى في فصول (`Chapters`).
- دعم رفع الملفات عبر النظام (`upload-file`).
- عرض محتوى فيديو، نصوص، وملفات PDF.

### 3. نظام الالتحاق والاشتراك
- تقديم طلب الالتحاق بالكورس.
- إدارة حالة الطلب: `Pending`, `Wait for Payment`, `Approved`.
- ربط الالتحاق بحالة الدفع.
- عرض الطلبات الخاصة بالطالب والمعلم.

### 4. الدفع الإلكتروني
- وحدة مدفوعات مفصّلة في الباك إند.
- دعم ربط حالة الدفع بحالة التسجيل.
- تسهيل دفع رسوم الكورسات للمستخدمين.

### 5. الجدول الدراسي وإدارة القاعات
- إدارة مواعيد الدروس والكورسات في الجدول.
- التحكم في القاعات وسعتها.
- منع تداخل المواعيد عبر التحقق من الجدول.

### 6. الحضور والغياب
- تسجيل حضور الطلاب لكل كورس.
- دعم الحالات: حاضر، غائب، متأخر.
- استخراج تقارير الحضور.

### 7. الواجبات والتقييم
- إنشاء واجبات للطلاب.
- رفع الحلول وتخزين ملفات الحل.
- تصحيح الواجبات وإضافة درجات وملاحظات.

### 8. نظام الاختبارات
- إنشاء اختبارات وأسئلة اختيار متعدد.
- حفظ خيارات الأسئلة وربطها بالاختبارات.
- تسجيل محاولات الطلاب ونتائجها.

### 9. تتبع التقدم
- حفظ تقدم الطالب داخل الفصول.
- حساب نسبة إنجاز الكورس.
- عرض التقدم بشكل مرئي في الواجهة الأمامية.

### 10. التواصل الداخلي والمراسلات
- دعم وحدة `chat` للمحادثات.
- إرسال إشعارات عبر البريد الإلكتروني (`mail`).

## 🧠 تقنية البنية (Architecture)

### Backend
- `src/` يحتوي على كود NestJS.
- وحدات رئيسية:
  - `auth`, `user`, `course`, `chapter`, `enrollment`, `chapter-progress`, `course-schedule`, `attendance`, `assignment`, `quiz`, `quiz-question`, `quiz-option`, `quiz-attempt`, `room`, `payments`, `chat`, `mail`, `upload-file`, `prisma`.
- `AppModule` يستورد الوحدات الأساسية للنظام.

### Frontend
- `front end/` يحتوي على مشروع React.
- يستخدم `React Router` و `Axios` للتواصل مع الباك إند.
- تصميم متجاوب ويدعم اللغة العربية RTL.

### قاعدة البيانات
- يستخدم `Prisma` للتعامل مع MySQL.
- ملفات المخطط والتسبيس في `prisma/schema.prisma`.
- بيانات مبدئية يمكن استيرادها من `mysql-init/`.

## ⚙️ كيفية التشغيل

### التشغيل باستخدام Docker (موصى به)
1. افتح Terminal في مجلد المشروع.
2. شغّل:
```bash
docker-compose up --build
```
3. افتح المتصفح:
- الواجهة الأمامية: `http://localhost:5173`
- الـ API: `http://localhost:3001`
- phpMyAdmin: `http://localhost:8080`

### إيقاف الخدمة
```bash
docker-compose down
```

### تشغيل محلي بدون Docker
1. باك إند:
```bash
npm install
npm run start:dev
```
2. واجهة أمامية:
```bash
cd "front end"
npm install
npm run dev
```

> تأكد من وجود قاعدة بيانات MySQL متصلة وتحديث إعدادات `.env` إذا لزم الأمر.

## 📂 هيكل المشروع
- `src/`: كود Back-end (NestJS)
- `front end/`: كود Front-end (React + Vite)
- `prisma/`: مخطط قاعدة البيانات وملفات الهجرات
- `docker-compose.yml`: إعداد حاويات المشروع
- `Dockerfile`, `Dockerfile.dev`: بناء بيئات العمل
- `mysql-init/`: ملفات إعداد قاعدة البيانات
- `public/`: موارد ثابتة مثل صور وفيديوهات وملفات PDF
